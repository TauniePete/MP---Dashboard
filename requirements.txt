# 📡 Sensor Traffic Analytics Dashboard

A polished, standalone Streamlit analytics dashboard for finalized sensor traffic monitoring data. Designed for executive review, operations analysis, and archival reporting.

---

## Overview

This dashboard visualizes a **finalized, closed dataset** of sensor traffic events collected during a completed monitoring project. No data uploads, authentication, or cloud services are required.

**Business rules already applied to the dataset:**
- Weekends removed
- Only 8:00 AM – 5:00 PM records retained
- Column J excluded permanently
- Dates formatted as long dates; times formatted as AM/PM

## Quick Start

### 1. Prerequisites

- Python **3.10 or higher**
- `pip` package manager

### 2. Clone or download the project

```bash
# If downloaded as a zip, extract it, then navigate to the folder:
cd sensor_dashboard
```

### 3. Create a virtual environment (recommended)

```bash
# macOS / Linux
python3 -m venv venv
source venv/bin/activate

# Windows
python -m venv venv
venv\Scripts\activate
```

### 4. Install dependencies

```bash
pip install -r requirements.txt
```

### 5. Run the dashboard

```bash
streamlit run app.py
```

The dashboard will open automatically at **http://localhost:8501** in your default browser.

## Sidebar Filters

All sidebar filters apply globally to Sections 2–6. The Executive Summary KPIs always reflect the **full unfiltered dataset** for accurate top-line reporting.

- **Date Range** — restrict analysis to a date window
- **Weekdays** — select specific days of the week
- **Sensor Type** — filter by Counter A / Counter B
- **Position** — filter by sensor position (Maindoor, 156, etc.)

Filters persist across all tabs during the session.

## Troubleshooting

### `ModuleNotFoundError`
Ensure your virtual environment is activated and dependencies are installed:
```bash
pip install -r requirements.txt
```

### `OSError: Data file not found`
Confirm `data/finalized_sensor_data.csv` exists inside the project folder.
Do **not** rename or move the file.

### Port already in use
Run on a different port:
```bash
streamlit run app.py --server.port 8502
```

### Slow initial load
The first load caches the dataset. Subsequent interactions are fast. This is expected behavior.

### Charts not rendering
Ensure `plotly>=5.22.0` is installed:
```bash
pip install --upgrade plotly
```

### Browser doesn't open automatically
Navigate manually to: **http://localhost:8501**

## Technical Notes

- **Caching**: `@st.cache_data` is applied to `load_data()` so the CSV is read only once per session.
- **Filtering**: Filters create a copy of the DataFrame — the original cache is never mutated.
- **Performance**: Optimized for datasets up to ~100k rows with Pandas groupby aggregations.
- **No database required**: All data lives in the single CSV file.
- **No internet required**: The dashboard runs fully offline after install.

## Sharing the Dashboard

### Option A — Streamlit Community Cloud (free public URL) ✅ Recommended

This is the easiest way to share a link anyone can open in a browser — no Python required on their end.

**Steps:**
1. Create a free account at [github.com](https://github.com) if you don't have one
2. Create a new **public** repository and upload the contents of this `sensor_dashboard/` folder
3. Go to [share.streamlit.io](https://share.streamlit.io) and sign in with GitHub
4. Click **New app** → select your repository → set main file path to `app.py` → click **Deploy**
5. In about 2–3 minutes you'll have a shareable URL like:
   `https://your-app-name.streamlit.app`

Anyone with that link can view the full interactive dashboard — no install needed.

> **Note:** Streamlit Community Cloud is free for public repositories. The data file (`data/finalized_sensor_data.csv`) must be included in the repository for the app to work.

### Option B — Send the ZIP file

Share the `sensor_dashboard.zip` file directly. The recipient will need Python installed and must follow the Quick Start steps in this README.

### Option C — Run on your local network

To let others on the same Wi-Fi or office network view the dashboard while it's running on your machine:

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

Then share your local IP address (e.g. `http://192.168.1.x:8501`). The dashboard is accessible as long as your machine is running it.

Best for: quick demos in the same office.

---

### Option D — Internal server / IT deployment

If your organisation has a server or cloud VM, the app can be deployed there permanently. Ask your IT team to run the app with:

```bash
streamlit run app.py --server.port 8501 --server.headless true
```

A reverse proxy (nginx or similar) can then expose it at a custom internal URL.
